> 大语言模型动辄几十层甚至上百层。这些层之间到底如何分工？分工是怎么形成的？为什么相邻层的 attention pattern 经常高度相似，却又不是在做重复计算？本文从 residual stream 的几何直觉出发，逐步建立一个"慢变路由 + 快变特征"的动力学图景，并借用物理学中重整化群（Renormalization Group, RG）的语言给出统一解释。在此基础上，我们讨论这个视角对下一代高效 LLM 架构的具体启发——包括如何理解 DeepSeek 式 sparse attention 的定位，以及"显式 semantic state"到底是必需组件还是可选增强。

## 1. 问题起点：层间分工从何而来？

### 1.1 现象：结构相同，功能却层次化

一个被反复观察到的现象是：大语言模型的不同层似乎存在某种分工。探针实验（probing）、可解释性研究和层消融（layer ablation）实验都指向类似的图景——

- **浅层**更多处理词面身份、词形变化、局部句法这类表层特征；
- **中间层**逐渐出现实体识别、指代消解、短语级语义组合；
- **深层**则更多承载实体间关系、事实关联、任务意图和最终的输出决策。

值得注意的是，Transformer 的每一层在结构上是完全相同的：同样的 Attention 模块、同样的 MLP、同样的残差连接。没有任何架构先验规定"第 3 层负责句法、第 20 层负责语义"。然而训练结束后，各层的功能却呈现出大致稳定的层次化。

这自然引出两个问题：**这种近似分工是如何形成的？背后的机制是什么？**

### 1.2 每一层实际在操作什么：residual stream

要回答这个问题，先看每一层实际在操作的对象。Transformer 的每一层通过残差连接更新一个共享向量，即 residual stream：

$$x_{l+1} = x_l + \Delta x_l$$

其中 $$\Delta x_l$$ 是第 $$l$$ 层 Attention 和 MLP 对当前表示的增量更新。注意这个结构的含义：**所有层读写的是同一个向量**。第 1 层写入的内容，第 30 层原则上仍然可以读到；第 15 层也完全可以覆盖或擦除第 5 层写入的东西。因此，层间分工必然体现为各层对这个共享向量的不同**使用方式**，而不是物理上的隔离。

### 1.3 候选解释：高维正交性？

一个看起来很自然的候选解释来自几何。residual stream 是高维向量（典型的 $$d$$ 在几千到上万），而高维空间有一个著名性质：两个随机独立向量几乎必然近似正交，

$$\frac{u^\top v}{\|u\|\,\|v\|} \approx 0$$

由此可以构造一个诱人的假说：既然高维空间能"容纳"大量近似正交的方向，不同层是否就是各自占据独立的正交子空间，互不干扰地写入不同层次的特征？如果是这样，"分工"就有了一个纯几何的解释——层与层之间天然不打架，各写各的。

### 1.4 为什么这个解释不成立

这个推断的漏洞在于前提。**高维正交性只对随机独立向量成立，而不同层的更新恰恰不是随机独立的**：

- 它们全部来自同一个输入序列；
- 它们由同一个训练目标（next-token prediction）端到端共同优化；
- 后一层的输入直接就是前一层的输出，存在强因果依赖；
- 训练出来的回路（circuit）经常刻意地读取、增强、删除或重写前面层写入的特征。

因此，层间的 residual update $$\Delta x_l$$ 与 $$\Delta x_{l'}$$ 之间的关系是多样的，取决于它们承担的计算角色：

- **近似正交**：两层写入互不相关的特征时；
- **高度正相关**：多层沿同一方向持续累积、逐步放大某个特征时（例如逐层强化一个答案方向）；
- **负相关**：某一层擦除前面层写入的、已经完成使命的中间特征时；
- **旋转/变换**：把一个已有特征读出来、变换成另一个特征再写回时。

换句话说，"每层一个正交子空间"把 residual stream 想象成了一排互不相通的抽屉，而实际情况更接近：

> **一个被所有层共享的高维工作空间（workspace）**——各层在其中读取、修改、增强、抑制和组合特征。

结论是：不同层确实存在统计性的功能分工，但正交子空间不是它的成因。真正的机制需要到别处去找。

## 2. 真正的机制：计算依赖决定计算阶段

### 2.1 高级特征依赖低级中间变量

"浅层局部、深层抽象"的趋势，主要不是几何决定的，而是**计算依赖关系**决定的。语言理解经常需要多步组合：

$$\text{token identity} \to \text{局部句法} \to \text{实体绑定} \to \text{关系表示} \to \text{输出决策}$$

以一个具体例子说明。要回答"文中提到的那位物理学家的导师是谁"，模型至少需要依次完成：识别哪些 token 构成人名（token → 实体）；确定"那位物理学家"指代的是哪个实体（指代消解，依赖实体表示）；在上下文中定位该实体与"导师"的关系（关系抽取，依赖实体绑定的结果）；最后把关系的另一端映射到输出（决策，依赖关系表示）。每一步的输入都是上一步的输出。**这种依赖链是逻辑上的串行结构，无法在单层内并行完成**，因此天然需要计算深度——而深度方向上的先后，就自然对应了功能上的浅/深分工。

### 2.2 一个容易混淆的区分：可访问性 vs 计算深度

有人可能会反驳：第一层的 attention 就是全局的，任何 token 都能直接"看到"任何其他 token，为什么还需要多层？

这里需要区分两件事。全局可访问性回答的是：

> **可以从哪里读取信息？**

而网络深度回答的是：

> **可以连续执行多少步组合计算？**

单层 attention 能把任意位置的信息搬运过来，但搬运不等于计算。一层 Attention + MLP 能执行的非线性组合是有限的；"识别实体 → 消解指代 → 抽取关系"这样的流水线，每一步都需要前一步已经**算完并写入** residual stream 的中间变量。能读到原始材料，不等于一步就能把菜做完。

### 2.3 重新表述层级分工

因此，层级 specialization 更准确的表述是：

> **计算阶段的分工，而不是严格语义子空间的分工。**

浅层之所以"处理"表层特征，是因为表层特征是依赖链的起点，必须先算；深层之所以"处理"抽象特征，是因为抽象特征依赖前面所有阶段的产出。分工是训练在计算依赖约束下自发组织出来的结果，而共享的 residual stream 则为各阶段之间传递中间变量提供了介质。

## 3. Residual Stream 的跨层信息传递：形成、休眠、重新激活

共享工作空间的一个重要推论是：信息可以跨越很多层进行"远距离"传递，而无需中间层参与。

残差连接提供了一条近似 identity 的通路。假设某个特征方向 $$v_f$$（比如"当前讨论的实体是某个人物"）在第 5 层被写入：

$$x_5 \supset \alpha\, v_f$$

只要第 6 到第 20 层的更新都不显著触碰这个方向（即 $$\Delta x_l$$ 在 $$v_f$$ 上的投影很小），这个特征就会沿着 identity path 被原样保留下来：

$$x_{20} \supset \alpha\, v_f$$

直到第 21 层某个 head 或 MLP 神经元恰好拥有与之对齐的读取方向 $$w$$，它才被重新激活并参与计算：

$$w^\top x_{20} \neq 0$$

于是一条信息在深度方向上可以经历完整的生命周期：

$$\text{形成} \to \text{休眠} \to \text{重新激活} \to \text{参与决策}$$

两点值得强调：

**第一**，这种"跨层跳跃"并不是物理上绕过中间层的捷径连接，而是共享工作空间的自然结果：某个 feature 被早期写入，长期保存，直到更深层才被再次调用。中间层对它而言只是"透明"的。

**第二**，这解释了为什么单看相邻两层的表示差异会低估模型内部的信息组织复杂度：一个在第 20 层突然"起作用"的特征，其来源可能远在第 5 层，中间的十几层与它毫无因果关系。这对可解释性研究（比如 activation patching 的层定位）和后面要讨论的架构设计（fine memory 是否值得保留）都有直接含义。

## 4. 一个实证现象：相邻层的 Attention 为什么如此相似

带着"共享工作空间 + 计算阶段"的图景，我们来看一个来自工程实践的实证观察。长上下文稀疏注意力的研究（其动机是复用 attention 计算以降低成本）反复发现：

- 相邻层的 Top-$$k$$ attention token 集合高度重叠；
- 高 attention score 的 token，其跨层稳定性尤其高；
- 完整的 attention 权重分布可能在层间变化，但**主要支持集**（谁进入了高分名单）变化很慢。

这个现象乍看有点反直觉——如果每层做的计算不同，为什么"看"的位置几乎一样？可以从四个层面解释。

### 4.1 相邻层 residual state 变化很小

在主流的 Pre-Norm Transformer 中，单层更新的范数通常远小于当前状态的范数：

$$\|\Delta x_l\| \ll \|x_l\|$$

因此相邻层的表示方向几乎不变：

$$\cos(x_l, x_{l+1}) \approx 1$$

而 attention 分数由 Q、K 决定，Q、K 又是 residual state 的（层内不同的）线性投影。当输入状态只发生小幅漂移时，即使投影矩阵不同，各 token 的相对语义关系——谁是实体、谁是证据、谁和谁相关——也不会在一层之内突变。

### 4.2 高分 token 对应稳定的语义关系

被分配高 attention 的 token 通常不是随机的，而是承担着结构性角色：

- 当前问题中的核心实体；
- 支持推理的相关证据句；
- 句子的主语或谓语；
- 代词的候选先行词；
- 最近一步推理的结论；
- 以及 attention sink（首 token）和局部窗口这类"结构性常客"。

这些关系反映的是输入本身的语义结构，而语义结构不会只维持一层的"寿命"。核心实体在第 10 层是核心实体，在第 15 层大概率还是。

### 4.3 Top-$$k$$ 集合比完整权重更稳定

这里有一个简单但重要的鲁棒性论证。考虑某 query 在相邻两层对四个 token 的打分：

$$s_l = [10,\; 8,\; 2,\; 1], \qquad s_{l+1} = [8,\; 10,\; 3,\; 0]$$

两层的分数明明发生了不小的变化——前两名甚至互换了位置——但 **Top-2 集合完全相同**。原因是高分组与低分组之间存在很大的 margin（8 vs 3）。只要这个 margin 存在，层间的分数扰动就只会改变高分 token 之间的相对排序和权重，而不会改变"谁在名单里"。因此，"Top-$$k$$ 集合稳定"是一个比"attention 权重稳定"弱得多、也更容易被满足的条件——这正是实证中观察到前者而非后者的原因。

### 4.4 Softmax 强化头部稳定性

Attention 权重经过 softmax：

$$a_j = \frac{\exp(s_j)}{\sum_m \exp(s_m)}$$

指数放大使得高分 token 拿走绝大部分权重，低分 token 的权重被指数压低。于是分数上的小扰动对"高分 token 是否主导"几乎没有影响——softmax 在功能上起到了对支持集的**锁定**作用。

## 5. 关键区分：信息来源相同 ≠ 信息处理相同

到这里出现了一个表面上的张力：第 2 节说不同层承担不同计算阶段，第 4 节又说相邻层"看"的位置几乎一样。相邻层到底是不是在做重复计算？

答案是否定的，而理解它的关键是把 Attention 拆成两个功能不同的回路。

### 5.1 QK circuit 与 OV circuit

**QK circuit** 决定"**从哪里读取信息**"。它由 query 和 key 的投影构成，产出 attention pattern：

$$A_l = \operatorname{softmax}\!\left(Q_l K_l^\top\right)$$

**OV circuit** 决定"**读取之后写入什么特征**"。它由 value 投影和输出投影构成，决定被读取的内容如何被变换后写回 residual stream：

$$\Delta x_l = A_l\, X_l\, W_V^{(l)} W_O^{(l)}$$

这两个回路的参数是完全独立的。因此即使相邻层的 attention pattern 近似相同：

$$A_l \approx A_{l+1}$$

它们的特征变换也完全可以不同：

$$W_V^{(l)} W_O^{(l)} \neq W_V^{(l+1)} W_O^{(l+1)}$$

### 5.2 同一批 token 上的流水线

于是一个自然的组织方式浮现出来：连续几层反复关注**同一批 token**，但对它们依次执行**不同的计算**：

$$\text{识别实体} \to \text{绑定关系} \to \text{形成事件表示} \to \text{映射到答案方向}$$

打个比方：几位工人围着同一张工作台（相同的 attention 支持集），第一位负责分拣原料，第二位负责组装，第三位负责质检——他们面前的材料是同一批，做的工序却完全不同。

> **相同信息来源，不代表相同信息处理。**

### 5.3 分工的自然粒度：routing regime

这个区分还带来一个关于"分工粒度"的推论。既然功能不同的相邻层可以共享同一个 attention 支持集，那么层级 specialization 的自然单位就可能不是单层，而是**共享同一路由结构的若干连续层**——不妨称之为一个 routing regime。在一个 regime 内部，模型对同一组相关 token 执行一段多步流水线；regime 之间的切换（attention pattern 的显著重组）则对应计算阶段的切换。这个概念在第 7 节的 RG 类比中会获得更精确的对应物。

## 6. 统一图景：慢变路由 + 快变特征

把第 4、5 节的观察合起来，可以把 Attention 的逐层动力学分解为两个时间尺度不同的成分：

**路由拓扑** $$A_l$$：决定 token 之间的通信结构——谁向谁传递信息。它在相邻层之间变化**较慢**，在计算阶段切换时发生重组。

**特征状态** $$x_l$$：token 所承载的具体特征内容。它在这张通信图上**持续快速演化**——每一层的 OV/MLP 都在改写它。

形式上即：

$$A_{l+1} \approx A_l, \qquad x_{l+1} \neq x_l$$

概括为一句话：

> **slow routing + fast feature dynamics**

通信图在若干层内保持近似稳定，特征沿着这张图不断 refinement。这个双时间尺度分解是理解 Transformer 深度方向动力学的核心图景——它同时解释了 attention 稳定性的实证观察（慢变量确实慢）和层间功能分工的存在（快变量确实在变）。而"一个系统中存在快慢分离的自由度，慢自由度定义有效结构、快自由度在其上演化"——这恰好是物理学中重整化群处理的标准情形。

## 7. 重整化群类比

### 7.1 RG 是什么：不是降维，而是识别 relevant 自由度

先为不熟悉的读者铺垫一下。重整化群是统计物理和场论中处理多尺度系统的框架，它回答的问题是：当我们从微观尺度走向宏观尺度时，哪些自由度的影响会被放大、哪些会消失，以及如何写出只包含"重要"自由度的**有效理论**。

RG 的核心不是简单的降维或平均，而是：

> **随着尺度变化，识别哪些自由度是 relevant 的，并构造新的有效描述。**

在 Wilson 的表述中，操作分三步。第一步，把场按频率分解为慢变和快变分量：

$$\phi = \phi_{\text{slow}} + \phi_{\text{fast}}$$

第二步，把快变（高频、短波长）自由度积分掉，它们的影响被吸收进慢变自由度的有效作用量：

$$e^{-S_{\text{eff}}[\phi_{\text{slow}}]} = \int \mathcal{D}\phi_{\text{fast}}\; e^{-S[\phi_{\text{slow}},\, \phi_{\text{fast}}]}$$

第三步，把有效作用量按一组算符基底展开：

$$S_l = \sum_i g_i^{(l)}\, \mathcal{O}_i$$

随着这个过程沿尺度反复进行，耦合常数 $$g_i^{(l)}$$ 发生"流动"（RG flow）：**relevant** 算符的耦合被逐步放大，主导宏观行为；**irrelevant** 算符的耦合被压低，微观细节被遗忘；**marginal** 算符则大致保持。流动的终点附近可能存在**固定点**（有效理论不再变化），不同流动区域之间的过渡称为 **crossover**。

RG 最深刻的洞见在于：宏观行为往往只由极少数 relevant 自由度决定，海量微观细节是 irrelevant 的——这正是"有效理论"之所以可能的原因。

### 7.2 Transformer 侧的对应展开

现在把 Transformer 的逐层表示放进同一个形式框架。借用可解释性研究中的 feature direction 观点，residual state 可以近似展开为一组特征方向的线性组合：

$$x^{(l)} \approx \sum_i a_i^{(l)}\, v_i$$

其中 $$v_i$$ 是 feature direction（类比算符基底 $$\mathcal{O}_i$$），$$a_i^{(l)}$$ 是该 feature 在第 $$l$$ 层的激活强度（类比耦合常数 $$g_i^{(l)}$$）。沿深度方向观察激活强度的演化，会看到三类行为：

- 某些 feature **随深度持续增强**——比如逐渐清晰的答案方向、任务意图——可类比 relevant operator；
- 某些 feature **基本保持**——比如贯穿全程都需要的实体身份——可类比 marginal operator；
- 某些 feature **逐渐衰减**——比如已完成使命的低级句法中间变量——可类比 irrelevant operator。

逐层的前向传播因此成为一种"流"：任务相关的有效自由度被逐步提炼和放大，无关细节被逐步压低。

### 7.3 完整对应表

| 重整化群 | Transformer |
|---|---|
| 微观自由度 | 原始 token / patch features |
| coarse-graining 操作 | Attention + MLP 的聚合与变换 |
| 有效场 | residual representation |
| 算符基底 $$\mathcal{O}_i$$ | feature directions $$v_i$$ |
| 耦合常数 $$g_i^{(l)}$$ | feature activations $$a_i^{(l)}$$ |
| relevant operator | 对输出越来越重要的 feature |
| irrelevant operator | 随深度被压低的细节 |
| RG flow | layer-wise representation flow |
| fixed point | 近似"输出充分"的表示（继续加层不再改变） |
| crossover | routing regime / 计算阶段的切换 |

最精炼的对应是：

> **RG 沿物理尺度提取低能相关自由度；Transformer 沿计算深度提取任务相关自由度。**

### 7.4 RG 语言下的 attention 稳定性

这套语言为第 4–6 节的现象提供了一个优雅的重述。相邻层高分 token 集合的稳定，可以理解为：

> **有效自由度已经稳定，但有效耦合仍在演化。**

具体地：QK circuit 选出的相关 token 集合，就是当前计算阶段的"有效变量集合"——它一旦确定就变化较慢；而 OV 和 MLP 不断改变这些 token 所承载的高层特征——这是有效耦合的持续流动。连续若干层因此构成一个**局部有效理论区间**：

$$\text{stable routing regime} + \text{gradually evolving effective features}$$

而当计算阶段切换、attention pattern 发生显著重组时，就类似 RG 流中的 crossover——系统从一个有效理论区间过渡到另一个，有效变量集合本身被重新选择。第 5.3 节提出的 routing regime，在这里获得了它的物理对应物。

## 8. 类比的边界：Transformer 不是严格的 RG

一个好的类比必须知道自己在哪里失效。以下四个差异不能忽略。

### 8.1 Transformer 通常不减少自由度数量

严格的 RG 每走一步都会真正减少自由度（block spin 合并格点，或动量壳积分削减模式数）。而标准 Transformer 逐层保持 token 数不变：

$$N_l = N_{l+1}$$

它对信息的"粗粒化"发生在表示内容上（无关特征被压低），而不是载体数量上。因此它更像一种**软 coarse-graining**，而非严格积分掉自由度。

### 8.2 微观旁路：被"积掉"的信息可以回来

这是最本质的差异。在严格的 Wilson RG 中，被积分掉的快自由度就永远消失了——它们的影响已被吸收进有效耦合，不会再作为显式变量出现。而 Transformer 的 residual stream 保留了微观旁路：如第 3 节所述，浅层写入的精细特征可以休眠数十层后被深层重新读取。模型可以在"已经很抽象"的深层，突然回头调用一个非常"微观"的细节（比如某个具体数字或变量名）。这种能力在物理 RG 中没有对应物，而它对语言任务至关重要——第 9 节会把它转化为一条架构原则。

### 8.3 每层参数不同：非自治流

RG 变换是同一个映射的反复迭代（这正是"群"这个名字的来源，尽管严格说是半群）。而 Transformer 每层参数独立：

$$F_l \neq F_{l+1}$$

因此逐层演化更像一个**非自治**（时变）动力系统，而不是同一变换的迭代。这既是差异也是自由度：模型可以为不同深度学出不同的"变换规则"。

### 8.4 相关性由任务定义，而非尺度变换

物理 RG 中，什么是 relevant 由尺度变换下的标度维数客观决定。而在 LLM 中，什么特征"重要"完全由训练目标——next-token prediction——定义。换一个任务分布，relevant/irrelevant 的划分就会不同。

综合这四点，更准确的名字是：

> **task-conditioned semantic RG**——一个由任务定义相关性、保留微观旁路、逐层规则可变的软粗粒化流。

## 9. RG 视角对架构设计的启发

如果 Transformer 隐式地在做 semantic RG，那么把这个过程**显式化**可能带来效率与能力上的收益。以下五个方向各自对应类比中的一个环节。

### 9.1 显式多尺度表示

标准 Transformer 的计算形状是均匀的：

$$N \times d \;\to\; N \times d \;\to\; \cdots$$

每一层都在全部 $$N$$ 个 token 上做全分辨率计算——但如果深层真正需要的有效自由度远少于 $$N$$，这就是浪费。可以改成序列长度递减、表示维度可变的显式层级：

$$(N_0, d_0) \;\to\; (N_1, d_1) \;\to\; (N_2, d_2), \qquad N_0 > N_1 > N_2$$

对语言而言，自然的尺度序列是：

$$\text{subword} \to \text{phrase} \to \text{entity/event} \to \text{task state}$$

即让架构本身承认"有效自由度随抽象层级递减"这一事实，而不是让每一层都背负全部微观 token 的计算成本。

### 9.2 Fine memory + coarse state

但 9.1 有一个陷阱：如果粗粒化是**不可逆**的——低层 token 被合并后彻底丢弃——就违背了第 8.2 节指出的关键能力：深层随时可能需要回读微观细节。语言任务中这种需求非常真实：精确的数字、人名、代码变量名，都无法从"语义大意"中重建。

因此正确的设计不是删除原始 token，而是把它们降级为**可检索的精细记忆** $$M_{\text{fine}}$$，供高层 latent 按需回读。整体形成：

> **coarse semantic state + fine retrievable memory**

粗粒度状态负责承载主线计算，精细记忆负责保证信息不可逆丢失不会发生——这正是把 residual 旁路从"隐式机制"升级为"显式架构组件"。

### 9.3 慢变路由的显式复用

第 4–6 节的观察有直接的工程含义：既然相邻层的 attention 支持集近似稳定（$$A_{l:l+k} \approx A_l$$），就没有必要每层都从头计算完整的 QK 打分。可以让若干连续层**共享或复用**同一个 attention support（一次路由决策，多层使用），各层只执行不同的 OV/MLP 更新。这样既保留了"同一批 token 上的多步流水线"这一计算模式，又省掉了冗余的路由计算——本质上是把 slow routing 的"慢"显式利用起来。

### 9.4 自适应深度：固定点判据

RG 流有固定点：当有效理论不再随尺度变化，流动就完成了。对应到 Transformer：当某个 token 或 latent 的表示已经接近收敛，

$$\|x_{l+1} - x_l\| < \epsilon$$

它就到达了自己的"固定点"——后续层对它不再产生实质更新。此时可以对它 early exit 或冻结，不再参与后续计算，形成 **token-wise adaptive depth**。直觉上也合理：一句话里的虚词可能几层就"算完"了，而承载关键推理的 token 需要走完全程。固定点判据为"每个 token 该算多深"提供了一个有原则的答案。

### 9.5 共享或半共享层参数：深度作为可学习的流

第 8.3 节指出 Transformer 是非自治流。反过来，也可以主动向"自治流"靠拢——让各层共享（或分组共享）参数，把深度写成显式的演化方程：

$$x_{l+1} = x_l + \Delta t\; F_\theta(x_l,\, s_l)$$

其中 $$s_l$$ 是可选的深度/阶段编码。这让深度方向更像一个真正可学习的连续 flow，参数量随深度解耦，也与 9.4 的自适应深度天然兼容（流走到固定点即停）。

### 9.6 语言的特征层级：不是树，而是三流结构

顺着多尺度的思路，语言中更自然的特征层级是：

$$\text{subword token} \to \text{phrase/span} \to \text{entity/event} \to \text{discourse state} \to \text{task/reasoning state}$$

但必须承认：语言的层级**不是固定的树状结构**——一个实体可以横跨多个句子，一条推理链可以跳跃地引用相距很远的证据。因此比"严格分层"更合理的组织方式是三个并行交互的流：

- **Token stream（persistent token memory）**：保存精确词面、数字、代码变量、引用位置、可复制内容——一切不能有损压缩的东西；
- **Semantic latent stream（dynamic semantic latents）**：维护 phrase、entity、event、relation、topic、evidence chunk 等动态语义变量；
- **Global reasoning stream（small global workspace）**：维护当前 query、plan、证据、中间状态、候选答案。

三个尺度之间双向交互：$$X \leftrightarrow Z \leftrightarrow G$$。低层为高层提供原材料，高层为低层的检索提供 query。

## 10. 案例分析：DeepSeek 式 Sparse Attention 是 RG 吗？

有了上面的框架，可以精确定位一个现实架构。DeepSeek V4 风格的 sparse attention 大致由三个访问通道组成：

- **Recent tokens**：最近若干 token 的完整精细上下文——全分辨率、无压缩；
- **CSA**（压缩稀疏检索）：对较远历史先做温和压缩形成索引，再由当前 query 选出少量相关位置做精读；
- **HCA**（高压缩全局通道）：对整个历史做高比例压缩，形成一个低成本、始终可见的全局视野。

从记忆结构看，这已经形成了三层组织：

$$\text{fine memory} + \text{compressed retrieval index} + \text{coarse global summary}$$

它的 RG 风格体现在两点：全局层面维护 coarse representation（粗粒化的有效描述），同时保留 query-conditioned 的精细检索路径（第 9.2 节的 fine memory 原则）——粗读保证视野，精读保证细节不丢。

但需要精确区分它回答了什么、没回答什么。Sparse attention 主要回答：

> **当前 query 应该读取哪些历史位置？**

而一个完整的显式 semantic hierarchy 还需要回答：

> **这些 token 组合起来构成了什么高层变量？**

前者只是在原有 token 图上做**边的稀疏化和访问的分层**——节点仍然是原始 token（或其压缩副本）；后者则要求产生**新的节点、新的状态、新的 feature basis**——比如一个显式的"实体状态"或"事件表示"，它不再是任何单个 token 的副本。用图的语言概括：

$$\text{sparse attention} \approx \text{稀疏边 + 分层访问}$$
$$\text{semantic hierarchy} \approx \text{新节点 + 新状态 + 新特征基}$$

所以严格地说，DeepSeek 式 sparse attention 层次化的是**访问方式**，而非**语义表示本身**。它是 semantic RG 的"读取侧"实现，粗粒化的"表示侧"仍然是隐式的。

## 11. Semantic State：必需组件还是可选增强？

顺着这个区分，一个自然的问题浮出水面：既然访问侧已经分层，是否应该再加一个显式的"表示侧"——在 recent tokens、CSA、HCA 之上，增加独立的第四层 semantic memory，显式维护实体、事件、任务状态？

### 11.1 默认答案：不必要

冷静地看，前三层已经覆盖了信息访问的完整谱系：

$$\text{局部细节（recent）} + \text{长程精确检索（CSA）} + \text{全局摘要（HCA）}$$

如果第四层的功能只是"对历史再做一次摘要"，那它与 HCA 重复，是冗余的。第四层只有在承担一个**性质不同的角色**时才有存在价值。这个角色是什么？

### 11.2 Retrieval 与 persistent state 是两种不同的功能

**Retrieval** 回答的是"**现在应该看什么**"：

$$q_t \;\to\; \{x_{j_1}, x_{j_2}, \ldots\}$$

它是无状态的：每次由当前 query 触发，从存储中取回内容，用完即弃。

**Persistent state** 回答的是"**到目前为止已经知道什么、任务进行到什么状态**"：

$$Z_t = f(x_{\le t})$$

它维护的是跨时间持续的变量——实体状态、已确认的约束、已做出的决策、部分结果、当前计划、尚存的不确定性——并随新输入增量更新：

$$Z_{t+1} = \operatorname{Update}(Z_t,\, x_{t+1})$$

前三层记忆全部属于 retrieval 侧（HCA 虽是摘要，但本质是静态压缩的可读内容）。Persistent state 是一个真正不同的东西。

### 11.3 它的真正价值：缓存计算，而不是缓存信息

关键的洞察是：persistent state 的价值不在于**存了什么信息**（信息前三层都有），而在于**存了什么计算结果**。

没有 persistent state 时，每次需要使用历史信息，都必须现场重走完整的推理流程：

$$\text{retrieve} \to \text{bind} \to \text{aggregate} \to \text{infer}$$

——检索相关片段、把它们绑定到正确的实体、聚合成一致的图景、再推出结论。哪怕上一步刚刚做过一模一样的推理，下一步也要从头再来，因为中间结果没有地方存放。

有了 persistent state，已完成的推理可以被缓存为一个状态：

$$z_t = f(x_{j_1}, \ldots, x_{j_k})$$

之后只需增量更新：

$$z_{t+1} = \operatorname{Update}(z_t,\, \Delta x)$$

这就是它的本质价值：

> **amortized reasoning / semantic computation cache**——把重复的多步推理摊销为一次构建加持续的小步更新。

### 11.4 什么时候值得、什么时候不值得

由此可以给出清晰的适用边界。

**值得的场景**——凡是"同一批中间结论会被反复使用、且随时间持续演化"的任务：

- 长程多步推理（中间结论需要跨很多步复用）；
- Agent（计划、已完成动作、环境状态需要持续追踪）；
- 代码执行与程序状态维护（变量绑定就是天然的 persistent state）；
- 长对话（用户偏好、已确认事实的累积）；
- 世界模型（实体与场景状态的持续演化）。

**不值得的场景**——凡是"current query + one-shot retrieval"就能解决的任务：

- 普通 next-token prediction；
- 短上下文问答；
- 单步事实检索、一次检索即可回答的问题；
- 当前 forward pass 内部的临时工作状态已经够用的任务。

在后一类任务上强行维护 persistent state，只是引入了冗余。

### 11.5 风险清单

显式高层状态还有不可忽视的代价：

- **错误累积**：状态一旦写错，会长期污染后续推理，且没有天然的纠错机制；
- **写入决策困难**：很难提前判断哪些信息值得持久化（这正是第二部分记忆讨论中的根本困境在此处的重现）；
- **状态漂移**：压缩后的状态可能逐渐偏离原文事实；
- **latent 幻觉**：高层 latent 可能"编造"从未发生的状态；
- **监督稀缺**：训练时对中间状态缺乏直接监督信号；
- **功能重复**：可能与 residual stream 内部已经隐式维护的工作状态冗余。

### 11.6 合理的设计：按需激活的 workspace，或 stateful HCA

综合收益与风险，合理的方案不是默认添加第四层，而是让它**按需激活**——用一个门控决定是否更新：

$$Z_{t+1} = (1 - g_t)\, Z_t + g_t\, \operatorname{Update}(Z_t,\, x_t)$$

简单任务时 $$g_t \approx 0$$，workspace 保持沉默、零开销；复杂长程任务时 $$g_t \approx 1$$，它作为计算缓存全速工作。

一个更简洁的实现方向甚至不需要新增模块：**stateful HCA**。既然 HCA 已经是一个始终可见的全局压缩表示，只需把它从"静态压缩 KV"升级为"可持续更新的 latent state"：

$$M_{t+1}^{\text{HCA}} = U\!\left(M_t^{\text{HCA}},\, x_{t+1}\right)$$

并配套几个机制：persistent slots（部分槽位专用于持久变量）、gated update（防止无差别覆写）、selective overwrite（只更新确实变化的槽位）、与 CSA 精细检索的配合（状态存结论，原文可回查）、confidence 与 provenance 标注（每条状态记录置信度和来源，缓解漂移与幻觉）。这样 HCA 一个组件同时承担"全局压缩记忆"和"语义状态"两个角色，架构复杂度不增加。

## 12. 统一图景与核心结论

### 12.1 架构图景

综合全文，未来高效 LLM 架构可以概括为四个组件的组合：

> **局部精细 token + 压缩全局记忆 + 稀疏按需检索 + 可选持续状态**

即：Recent Fine Tokens + CSA 式稀疏检索 + Stateful HCA / 全局 latent + Optional Persistent Workspace。各组件分工明确：recent tokens 保存精确局部信息；CSA 负责按 query 找回相关历史细节；HCA 提供低成本全局视野；persistent workspace 只在需要长期累积计算状态时启用。

从 RG 角度，这个组合对应一句话：

> **微观 token 被保留在可检索 memory 中，任务相关的有效自由度被提升到 coarse latent 中，并沿深度和时间继续演化。**

### 12.2 核心结论

1. **关于层间分工**：不同层主要承担不同的计算阶段，而不是占据严格正交的子空间。分工由计算依赖关系塑造，residual stream 是各阶段共享的工作空间。
2. **关于 attention 稳定性**：相邻层的信息来源（高分 token 集合）可以稳定，但特征变换（OV/MLP）仍在持续进行——QK 与 OV 的分工使"同一批 token 上的多步流水线"成为可能。
3. **关于跨层跳跃**：residual stream 允许特征被早期写入、长期休眠、深层重新调用——这是与严格 RG 最本质的差异，也是 fine retrievable memory 这条架构原则的来源。
4. **关于 RG 类比**：Transformer 沿计算深度提取任务相关的有效自由度，是一个 task-conditioned semantic RG——软粗粒化、保留微观旁路、逐层规则可变、相关性由任务定义。
5. **关于 sparse attention**：它层次化的是访问方式（稀疏边、分层读取），而非完整的语义表示（新节点、新状态）——是 semantic RG 的读取侧实现。
6. **关于 semantic state**：它不是 memory hierarchy 的必需组成，而是 stateful computation 的可选增强；其真正价值是 amortized reasoning，合理形态是按需激活的 workspace 或 stateful HCA。

### 12.3 最值得探索的方向

> **slow routing + fast feature dynamics + fine retrievable memory + stateful coarse latents**

——让路由的慢、特征的快、微观记忆的可回读、粗粒状态的可持续，各自成为显式的架构一等公民。
